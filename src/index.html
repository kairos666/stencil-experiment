<waf-input>

text input field decorator to add material design v1 look & feel to any <input> tag

Do ot interfere with native input attributes. ex: type="email", pattern, placeholder, value, disabled, required, ...

minimal requirement

  • id attribute must be provided on <input> tag

limitations

  • not fit for custom sync|async validation, need to be extended or using another component
  • input attributes are only evaluated at bootstrap time, if "placeholder" value is changed, for example, it is ignored by the <waf-input> component, you would have to recreate a new one
  • custom specifications are tied to theme and may overload the component uselessly for other themes

<waf-img>

replacement for <img> tag. Accepts all attributes that a regular img tag would accept (ignoring deprecated ones). Only add custom loading enhancement for loading images asynchroneously thus improving performance of the page first paint.

Add max-width:100% on images by default to avoid breaking layout in RWD.

minimal requirement

  • src attribute must be provided on waf-img tag

optional requirements

  • alt attribute for textual alternative to image (optional because accessibility requires to not have them in case of a purely decorational visual)
  • visible-async-loading boolean attribute that change behavior to async load the image if both the document is ready AND the waf-img is visible in the browser
  • srcset regular <img> attribute
  • decoding regular <img> attribute
  • width regular <img> attribute. Loading placeholder apply also the specified width.
  • height regular <img> attribute. Loading placeholder apply also the specified height.
  • sizes regular <img> attribute

limitations

  • can't specify custom HTML for loading and broken placeholders. You can still style them though.
  • not optimal for srcset settings because the browser act as a black box and we don't know which image source will be applied beforehand.
  • <waf-img> can't be self closing tag like <img> because it is not allowed by CustomElements specs

<waf-dialog>

replacement for <dialog> tag due to scarce support in browsers.

Dead simple, accessible, responsive ready dialog boxes

minimal requirement

No minimal requirement, just put the tag around the content you want to serve in a dialog box.

3 slots are available to distribute your content across the dialog box. You can target those with the slot attribute. Possible values are: title, content, actions

optional requirements

  • limited-height enable the modal to be completely visible on screen, only the content part is scrolling if too much content. Without this attribute, the full page scrolls for revealing a long dialog box.
  • no-backdrop boolean attribute that remove the backdrop effect on the page when dialog box is open.
  • prevent-backdrop-closing boolean attribute that remove the ability to click the backdrop to close the dialog. Useful to force user to interact with the dialog box. Escape is still working though (accessibility)
  • data-dialog-close attribute you can put on any element contained inside the tag. When a user click this element it will close the dialog.

limitations

  • There is no built-in behavior where you make sure only one opened modal at any given time. It is up to the developer to manage that.
  • IE11 has issues rendering the limited-height modal when there is lots of content. It is a known issue, having a relative unit for flex-basis trigger the bug. Setting an absolute value like "350px" fix the issue. But because this has to be calculated dynamically this is out of scope of the component.
  • Not really a good idea to nest these tags inside one another.

<waf-tabs>

Easily generate, accessible, good looking, nestable, material design tabs

minimal requirement

  • One <waf-tabs> containing exclusively <waf-tab> tags (one for each tab)
  • Each <waf-tab> tag has a title attribute (value is either string or HTML string)
  • Just insert your content inside <waf-tab> for each pane

optional requirements

  • selected boolean attribute allows to declare which pane will be opened initially (default if not set is first). Only set this once per tabs system.

limitations

  • No advanced RWD behavior, this means that if you have many tabs the tab system will not look nice with a narrow screen

<waf-utils-ripple-effect>

Apply material design ripple effect on your buttons. Act like narrowly scoped behavior & style theme

minimal requirement

No minimal requirement, just put the tag around the content you want to target for the effect.

By default, all DOM elements with the ripple attribute will have the effect attached.

optional requirements

  • selector enables to provide a CSS selector string to target specific elements. effect will be applied to either matching element, and elements with ripple attribute

limitations

  • Effect can be hindered by host styles
