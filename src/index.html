WAF-INPUT - textfield

Un champs texte qui englobe un tag input et le style à la Material design. Le champs <input> tag reste complètement fonctionnel, il est simplement "décoré" avec un label et un style approprié. Les erreurs natives sont également gérées (type="email", pattern, placeholder, value, disabled, required, ...).

<waf-input> API - minimal requirement

  • insert a unique input tag inside
  • "label" attribute must be provided on waf-input tag (accessibility binding with id is automatic)
  • "id" attribute must be provided on input (would not make sense to auto generate it)

<waf-input> tag API - optional requirement

  • float (mdl specific) boolean attribute to customize behavior with floated animated label
  • alignRight (mdl specific) boolean attribute to customize style for RTL languages
  • fullWidth (mdl specific) boolean attribute to customize style for full width inputs
  • disabled|required|pattern|value|step|placeholder|... accept and react to all valid <input> attributes without alterations
  • errors you can provide a set of custom error messages to override the built-in defaults

<waf-input> tag API - limitations

  • not fit for custom sync|async validation, need to be extended or using another component
  • input attributes are only evaluated at bootstrap time, if "placeholder" value is changed, for exmaple, it is ignored by the <waf-input> component, you would have to recreate a new one
  • custom specifications are tied to theme and may overload the component uselessly for other themes

WAF-IMG - image async loading

replacement for <img> tag. Accepts all attributes that a regular img tag would accept (ignoring deprecated ones). Only add custom loading enhancement for loading images asynchroneously thus improving performance of the page first paint.

Add max-width:100% on images by default to avoid breaking layout in RWD.

<waf-img> API - minimal requirement

  • "src" attribute must be provided on waf-img tag

<waf-img> tag API - optional requirement

  • alt attribute for textual alternative to image (optional because accessibility requires to not have them in case of a purely decorational visual)
  • visible-async-loading boolean attribute that change behavior to async load the image if both the document is ready AND the waf-img is visible in the browser
  • srcset regular <img> attribute
  • decoding regular <img> attribute
  • width regular <img> attribute. Loading placeholder apply also the specified width.
  • height regular <img> attribute. Loading placeholder apply also the specified height.
  • sizes regular <img> attribute

<waf-img> tag API - limitations

  • can't specify custom HTML for loading and broken placeholders. You can still style them though.
  • not optimal for srcset settings because the browser act as a black box and we don't know which image source will be applied beforehand.
  • <waf-img> can't be self closing tag like <img> because it is not allowed by CustomElements specs
